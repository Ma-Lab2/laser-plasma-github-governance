New Member Setup

给“新号 + 新 WSL 电脑”一条连续接入路径

这页专门解决第一次接入课题组的现实问题：账号怎么注册、组织怎么加、WSL 里先装什么、SSH 怎么配、Codex 怎么验证、治理仓库怎么拉、skill 怎么装。

直接按步骤执行 如果你要先配 AI Agent
Do Not Skip

不要先 clone 再补配置

最常见的返工就是：没配 SSH 就开始 clone，没搞清楚 WSL 和 Windows 的环境差异就开始让 agent 帮忙。先把环境和认证做对，后面会省很多时间。

Sequence

首次接入顺序

01

注册 GitHub

使用常用邮箱注册账号，验证邮箱，建议开启双因素认证。

02

加入 Ma-Lab2

由管理员发组织邀请，你在邮箱或 GitHub 通知里接受。没有加入组织，就算知道仓库地址也可能没有访问权限。

03

进入 WSL，检查基础命令

必须确认自己真正跑在 WSL 里，再检查 git、python3、ssh。

uname -a
command -v git
command -v python3
command -v ssh
04

配置 SSH

优先使用 SSH 进行日常 Git 传输。如果已有公钥，先检查是否已上传 GitHub；如果没有，再生成新的。

ls -la ~/.ssh
ssh-keygen -t ed25519 -C "your-github-email@example.com"
cat ~/.ssh/id_ed25519.pub
ssh -T git@github.com
05

安装并验证 Codex / codex cli

安装方式可能因机器不同而不同，但在组内协作前至少要确认命令存在、版本可读、可登录。

command -v codex
codex --version
codex login
06

用 SSH 克隆治理仓库

治理仓库是课题组协作标准入口。不要先去业务仓库硬上手。

git clone git@github.com:Ma-Lab2/laser-plasma-github-governance.git
cd laser-plasma-github-governance
07

运行自检并安装 skill

先让脚本检测 runtime、SSH、token 和仓库状态，再安装审计 skill。

./scripts/onboarding-check.sh --org Ma-Lab2 --repo laser-plasma-github-governance
./scripts/install-governance-audit-skill.sh
Auth Split

SSH 和 GH_TOKEN 的职责分工

SSH：给 Git 日常传输

  • git clone
  • git pull
  • git push

推荐在课题组日常协作中默认使用 SSH。

GH_TOKEN：给治理脚本和 API

  • 组织审计
  • 治理脚本
  • GitHub API 调用

只在本机环境变量中配置，不要发给聊天里的 agent。

source ~/.bash_profile
echo ${GH_TOKEN:+SET}
