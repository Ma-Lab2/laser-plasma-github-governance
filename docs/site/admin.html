Admin Governance

管理员的工作是把项目变成可接管系统

管理员不只是“最后点 merge 的人”。你要保证仓库有治理模板、分支保护、required checks、版本与 changelog、至少两名维护者，以及毕业后的交接能力。

Admin Goal

核心不是短期跑起来，而是长期不断代

如果一个项目只有一名学生知道怎么继续维护，它其实还没有达到课题组级别的治理标准。

Repository Baseline

新仓库初始化

安装治理模板

./scripts/apply-governance-template.sh /path/to/target-repo 0.2.1
python3 .governance/update-skill-lock.py --skill-version 0.2.1

替换占位符和建立初始版本

至少更新 OWNERS.yaml、CODEOWNERS、docs/HANDOVER.md、CONTRIBUTING.md，并在仓库首次稳定后打 v0.1.0 标签。

Branch Protection

分支保护最小要求

禁止直接 push 默认分支

哪怕成员有写权限，也应该通过 PR 合并，而不是直接往 main 写。

至少 1 个 review approval

把代码合并的决定从“谁手快”变成“谁审过”。

required checks

governance/validate-pr-fields 和 governance/validate-version 至少要启用。

Audit

组织级审计

先看全量报告

./skills/governance-audit/scripts/run-org-audit.sh \
  --org Ma-Lab2 \
  --fail-on-blocker false

修完后再阻断验证

./skills/governance-audit/scripts/run-org-audit.sh \
  --org Ma-Lab2 \
  --fail-on-blocker true
Handover

毕业交接最小要求

至少两名维护者

不能让项目只在单人脑子里运行。

保留 HANDOVER 文档

后续接手者要能知道运行方式、关键路径、版本状态和风险点。
